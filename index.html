What this tool does

        
This tool helps convert client-provided email HTML into ready-to-use branded marketing output.

        
You can paste a full exported email or a content block and quickly turn it into either:

        
    
          
  • a fully branded email, or
    • 
          
  • a clean content fragment for use inside another email template.
    • 
        

        
The goal is to preserve as much client layout as possible while applying brand structure, cleanup rules, preview tools, and QA checks.

      


      

        
Output modes

        
Full branded email

        
Use this when you want a complete email with the selected brand header and footer applied.

        
In this mode the tool:

        
    
          
  • can preserve the client’s full email structure when needed
    • 
          
  • inserts the selected brand header and footer
    • 
          
  • applies standardized brand wrapping used across supported publications
    • 
          
  • supports tracking and stealth link injection
    • 
          
  • Runs QA checks to catch common email issues
    • 
        

        
This is the main workflow for sponsored sends and other complete branded emails.

        
Code fragment

        
Use this when you only need the content block, not a full email.

        
Fragment mode:

        
    
          
  • cleans up the pasted HTML
    • 
          
  • can preserve embedded styles for responsiveness
    • 
          
  • outputs only the cleaned content section
    • 
        

        
Fragments cannot be sent on their own. They must be inserted into another email template, such as a template built in the Omeda Email Designer.

      


      

        
Typical workflow

        
    
          
  1. Paste HTML into the input area or drag in an `.html` file.
    2. 
          
  2. Choose your output mode:
            
      
          
    • Full branded email for a complete email
      • 
          
    • Code fragment for content used inside another template
      • 
            
    
          
    3. 
          
  3. Select a brand when building a full email.
    4. 
          
  4. Review the preview using Desktop, Mobile, Compare, Code, and Dark mode.
    5. 
          
  5. Check any QA warnings.
    6. 
          
  6. Copy or export the finished HTML.
    7. 
        

      


      

        
Preview and code view

        
The preview lets you check how the email will appear before exporting.

        
You can switch between:

        
    
          
  • Desktop view
    • 
          
  • Mobile view
    • 
          
  • Compare view
    • 
          
  • Dark mode
    • 
          
  • Code view with editable output, live preview, revert, and find
    • 
        

        
This helps catch layout issues early and gives you a place to make final code edits without leaving the tool.

      


      

        
QA checks

        
Before exporting, the tool scans the HTML for common email risks such as:

        
    
          
  • missing image alt text
    • 
          
  • missing, placeholder, malformed, or non-HTTPS links
    • 
          
  • oversized email files
    • 
          
  • unsupported or risky HTML
    • 
          
  • Outlook rendering issues
    • 
          
  • formatting problems that could break layouts
    • 
        

        
The QA drawer separates fails and passes so you can review issues quickly without losing sight of what already looks good.

      


      

        
Tracking and settings

        
The tool also includes options for:

        
    
          
  • Ad ID tracking
    • 
          
  • default stealth links in Full branded email, or manual stealth links when added
    • 
          
  • cleanup controls by output mode
    • 
          
  • header and footer color settings for full branded email
    • 
        

        
These settings are there when needed, but the default workflow should still work well for most sends.

      


      

        
Helpful tips

        
    
          
  • Most workflows should use Full branded email mode.
    • 
          
  • If a client gives you a complete exported email, turning on Preserve full client template is often the safest full-mode choice.
    • 
          
  • Use Code fragment only when inserting content into an existing email template.
    • 
          
  • Use Code view when you need to make final output edits without changing the original pasted source.
    • 
          
  • Always check mobile, code, and QA before exporting.
