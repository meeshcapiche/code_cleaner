What this tool does

This tool helps convert raw email HTML into ready-to-use branded marketing output.

Instead of manually editing code, you can paste HTML from a vendor, partner, or content hub and the tool will quickly convert it into either:

  • a fully branded email, or
  • a clean content fragment for use inside another email template.

The tool keeps most of the original content structure intact while applying brand structure, improving stability, and checking for common email issues.

Output modes

Full branded email

Use this when you want a complete email with the correct brand header and footer automatically applied.

In this mode the tool:

  • Preserves most of the HTML you paste
  • Inserts the selected brand header and footer
  • Applies standardized brand wrapping used across supported publications
  • Makes small structural adjustments that help emails render more reliably
  • Runs QA checks to catch common email issues

This is the most common workflow and is typically used when preparing full marketing emails or sponsored sends.

Code fragment

Use this when you only need the content block, not a full email.

Fragment mode:

  • Cleans up the pasted HTML
  • Removes unnecessary or unsupported code
  • Outputs only the content section

Fragments cannot be sent on their own. They must be inserted into another email template, such as a template built in the Omeda Email Designer.

Typical workflow

  1. Paste HTML into the input area or drag in an `.html` file.
  2. Choose your output mode:
    • Full branded email for a complete email
    • Code fragment for content used inside another template
  3. Select a brand when building a full email.
  4. Review the preview using Desktop, Mobile, Code, and Dark mode.
  5. Check any QA warnings.
  6. Copy or export the finished HTML.

What the preview shows

The preview lets you quickly check how the email will appear before exporting.

You can switch between:

  • Desktop view
  • Mobile view
  • Dark mode
  • Code view

This helps catch layout issues early, before the email is copied into another system.

QA checks

Before exporting, the tool scans the HTML for common email risks such as:

  • oversized email files
  • unsupported or risky HTML
  • Outlook rendering issues
  • formatting problems that could break layouts

These warnings help flag likely issues early, but they do not replace final review.

Advanced tools

The Advanced section includes optional settings for:

  • ad tracking inputs
  • stealth links
  • additional code cleanup

These tools are available for more complex workflows but are separate from the default branded-email process so they don’t accidentally affect normal production.

Helpful tips

  • Most workflows should use Full branded email mode.
  • Use Code fragment only when inserting content into an existing email template.
  • Always check mobile and dark mode previews before exporting.
  • QA warnings highlight potential issues but final review is still recommended.
